JavaScript对象继承方法详解

在JavaScript中，几乎所有对象都继承自Object.prototype，这意味着它们都拥有Object.prototype上定义的方法。这些方法为所有对象提供了基础功能，使得我们可以对对象进行各种操作和检查。

理解这些继承方法对于掌握JavaScript对象模型至关重要，它们是JavaScript语言的核心组成部分。

hasOwnProperty()

检查对象自身是否具有指定的属性（不包括继承的属性）。

代码示例：
const obj = {
  name: '张三',
  age: 25
};

// 添加继承属性
obj.__proto__.inheritedProp = '继承属性';

console.log(obj.hasOwnProperty('name')); // true
console.log(obj.hasOwnProperty('inheritedProp')); // false
console.log(obj.hasOwnProperty('toString')); // false
注意事项：

该方法只检查对象自身的属性，不包括原型链上的属性。这对于区分对象自身属性和继承属性非常有用。

toString()

返回一个表示该对象的字符串。默认返回"[object Type]"格式。

代码示例：
const obj = { name: '测试对象' };
const arr = [1, 2, 3];
const func = function() {};

console.log(obj.toString()); // "[object Object]"
console.log(arr.toString()); // "1,2,3"
console.log(func.toString()); // "function() {}"
注意事项：

许多内置对象（如Array、Function）都重写了toString()方法以提供更有意义的字符串表示。可以自定义toString()方法来改变对象的字符串表示。

valueOf()

返回对象的原始值。默认情况下返回对象本身。

代码示例：
const obj = { value: 42 };
const num = new Number(10);
const date = new Date();

console.log(obj.valueOf()); // { value: 42 }
console.log(num.valueOf()); // 10
console.log(date.valueOf()); // 时间戳
注意事项：

与toString()类似，许多内置对象重写了valueOf()方法。当对象需要转换为原始值时（如数学运算），JavaScript会自动调用此方法。

isPrototypeOf()

检查一个对象是否存在于另一个对象的原型链上。

代码示例：
const parent = { a: 1 };
const child = Object.create(parent);

console.log(parent.isPrototypeOf(child)); // true
console.log(Object.prototype.isPrototypeOf(child)); // true
console.log(Object.prototype.isPrototypeOf(parent)); // true
注意事项：

该方法用于检查原型链关系，是理解JavaScript继承机制的重要工具。Object.prototype.isPrototypeOf()可以检查任何对象是否为JavaScript对象。

propertyIsEnumerable()

检查指定的属性是否为可枚举属性（即是否可通过for...in循环遍历）。

代码示例：
const obj = { name: '张三', age: 25 };
Object.defineProperty(obj, 'hidden', {
  value: '隐藏属性',
  enumerable: false
});

console.log(obj.propertyIsEnumerable('name')); // true
console.log(obj.propertyIsEnumerable('hidden')); // false
console.log(obj.propertyIsEnumerable('toString')); // false
注意事项：

默认情况下，对象的属性都是可枚举的，但可以通过Object.defineProperty()设置enumerable为false来创建不可枚举属性。

__proto__

指向对象的原型（内部[[Prototype]]属性）。这是一个非标准但广泛支持的属性。

代码示例：
const obj = {};
const arr = [];

console.log(obj.__proto__ === Object.prototype); // true
console.log(arr.__proto__ === Array.prototype); // true
console.log(Array.prototype.__proto__ === Object.prototype); // true

// 修改原型（不推荐）
obj.__proto__ = { customMethod: function() { return '自定义方法'; } };
console.log(obj.customMethod()); // "自定义方法"
注意事项：

__proto__是非标准属性，虽然被大多数浏览器支持，但不推荐在生产环境中使用。应使用Object.getPrototypeOf()和Object.setPrototypeOf()替代。
